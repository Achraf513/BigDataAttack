Semantic Url Attacks

In a semantic URL attack, a client manually adjusts the parameters of its request by maintaining the URL's syntax but altering its semantic meaning.

So what if a website is not that secured, and it does not implement good authentication and authorization solutions ?

you guessed it, an attacker can gain full access to private and confidential data, which has never been displayed to the outside world.





Our Solution ?

our solution is entirely based on the fact that the action of manually adjusting the parameters of a request's URL, is considered as a malicious behavior

The website will be collecting every requests data, including ip address, browser, version, OS etc .. and more importantly the URL route, the rest of the data is used to track back the attacker


We developed an AI model based on anomaly detection, in order to study the behavior of each request, understand the normal routes of a website ( the solution is dynamic ) and as a final step provides decisions to the routing system of the website ( attack or normal behavior )
